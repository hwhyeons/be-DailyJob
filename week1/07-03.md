# be-DailyJob 

> 이 회고는 작업량을 보고하기 위한 문서가 아니라, 오늘 내가 어떤 판단을 했고 어떤 개발 행동을 반복하거나 바꿔야 하는지 확인하기 위한 기록입니다. 
> 각 항목은 길게 쓰기보다, 사실/판단/근거/다음 행동이 드러나게 구체적으로 작성하세요.

###### 1. 오늘 완료한 것
- 오늘 실제로 끝낸 기능/수정/학습: 다양한 컨텐츠 타입 지원하는 웹서버 구현
- (오늘의 목표 중에서) 아직 끝내지 못한 부분: 3단계 구현

###### 2. 오늘 가장 의미있었던 기술적 판단
- 내가 내린 판단: Request Header의 첫번째 줄에서 요청한 리소스의 확장자를 검사해서 Response 타입을 결정함
- 그렇게 판단한 이유: HTTP Request를 보낼 때, Content-Type을 안보내는 경우도 있다는 것을 확인함
- 고려했지만 선택하지 않은 대안: Response Header에 Content-Type을 결정하기 위해 Request Header에 있는 Content-Type을 직접 읽는 방식이였음

###### 3. 오늘 잘한 행동(개발과 관련된 것)
- 어떤 행동이었는지: Git의 동작 원리(Staging Area, Working Directory, Reset/Revert)부터 Spring Boot의 컨트롤러 vs 정적 리소스 처리 우선순위, Java switch expression 등 모호하게 넘어가기 쉬운 프레임워크 및 언어의 내부 메커니즘을 깊이 있게 파고들어 정리한 것
- 그 접근이 도움이 된 이유: 단순히 코드를 임시방편으로 수정하는 데 그치지 않고 시스템의 작동 원리를 명확히 이해함으로써, 향후 발생할 수 있는 잠재적 에러(예: 정적 파일 매핑 충돌, Git 커밋 꼬임 등)를 근본적으로 예방하고 스스로 디버깅할 수 있는 단단한 기초를 쌓았음
- 다음에도 반복하고 싶은 점: 새로운 도구나 문법을 접할 때 기능 사용법만 익히고 넘어가는 것이 아니라, "내부적으로 왜 이렇게 동작하는지" 구조와 원리를 찾아보고 내 것으로 만드는 학습 태도 유지하기.
