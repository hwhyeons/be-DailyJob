# be-DailyJob

> 이 회고는 작업량을 보고하기 위한 문서가 아니라,
오늘 내가 어떤 판단을 했고 어떤 개발 행동을 반복하거나 바꿔야 하는지 확인하기 위한 기록입니다.
각 항목은 길게 쓰기보다, 사실/판단/근거/다음 행동이 드러나게 구체적으로 작성하세요.
> 

### 1. 오늘 완료한 것

- 오늘 실제로 끝낸 기능/수정/학습
    - 세션 기반 로그인
    - 로그아웃

### 2. 오늘 가장 의미있었던 기술적 판단

- 내가 내린 판단
    - 로그인 성공시에, 메인페이지(/index.html)로의 이동을 JavaScript를 이용해서 이동
- 그렇게 판단한 이유
    - 로그인 과정에서, 이미 fetch()를 통해서 작업을 하고 있었고,
    기존의 Form을 이용한 POST 방식을 fetch 방식으로 변경했었음
- 고려했지만 선택하지 않은 대안
    - 처음에는 Redirect 302를 이용해서 페이지를 이동시키려함

### 3. 가장 막혔던 문제와 해결 과정

- 문제 상황 또는 증상
    - 서버에서 HTTP Response를 302로, Location: /index.html로 해놨는데
    거기로 이동을 안하고 다른페이지로 이동함
- 처음 추측한 원인
    - HTTP Response Header의 Location 잘못 작성
- 원인을 확인하기 위해 시도한 방법
    - 디버그 모드로 한줄 한줄 진행하면서 확인
- 실제 원인
    - HTTP Response를 302로, Location: /index.html은 제대로 설정했지만,
    자바스크립트에서 `/index.html`이 아닌 `/`로 이동시키고 있었음
- 해결 방법
    - 302 Response를 제거하고 자바스크립트로 `/index.html`로 이동하게 설정
